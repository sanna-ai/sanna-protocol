# @sanna-ai/core

TypeScript SDK for the [Sanna protocol](https://github.com/sanna-ai/sanna-protocol) ‚Äî an open AI governance framework that enforces constitutional boundaries on AI agents through cryptographic receipts. Constitutions define what an agent can and cannot do; receipts provide tamper-evident proof that every action was evaluated against those rules.

Implements **Sanna Protocol v1.3**.

## Install

```bash
npm install @sanna-ai/core
```

Requires Node.js 22+ (native Ed25519 support).

## Quick Start

```typescript
import {
  loadConstitution,
  verifyConstitutionSignature,
  loadPublicKey,
  loadPrivateKey,
  evaluateAuthority,
  generateReceipt,
  signReceipt,
  verifyReceipt,
} from "@sanna-ai/core";

// 1. Load and verify a constitution
const constitution = loadConstitution("constitution.yaml");
const authorKey = loadPublicKey("author.pub");
const valid = verifyConstitutionSignature(constitution, authorKey);

// 2. Evaluate an action against the constitution
const decision = evaluateAuthority(
  "database.write",
  { table: "users", operation: "delete" },
  constitution,
);
// decision.type ‚Üí "can_execute" | "cannot_execute" | "must_escalate"

// 3. Generate a receipt
const receipt = generateReceipt({
  correlation_id: "req-001",
  inputs: { query: "Delete user 42", context: "admin panel" },
  outputs: { response: "User deleted" },
  checks: [
    { check_id: "auth-check", passed: true, severity: "critical", evidence: null },
  ],
});

// 4. Sign the receipt
const privKey = loadPrivateKey("author.key");
signReceipt(receipt as unknown as Record<string, unknown>, privKey, "agent@example.com");

// 5. Verify a receipt
const result = verifyReceipt(receipt as unknown as Record<string, unknown>, authorKey);
// result.valid ‚Üí true
// result.checks_performed ‚Üí ["schema", "signature", "fingerprint", "content_hashes", ...]
```

## API Reference

### Hashing

- `canonicalize(obj)` ‚Äî RFC 8785 JSON Canonicalization Scheme
- `hashBytes(data)` ‚Äî SHA-256 of raw bytes (hex string)
- `hashContent(data, truncate?)` ‚Äî SHA-256 with NFC + line-ending normalization
- `hashObj(obj)` ‚Äî canonicalize then SHA-256
- `EMPTY_HASH` ‚Äî SHA-256 of empty input

### Crypto

- `generateKeypair(label?)` ‚Äî generate Ed25519 keypair
- `sign(data, privateKey)` ‚Äî Ed25519 sign, returns base64
- `verify(data, signature, publicKey)` ‚Äî Ed25519 verify
- `loadPrivateKey(path)` ‚Äî load PKCS#8 PEM private key
- `loadPublicKey(path)` ‚Äî load SPKI PEM public key
- `getKeyId(key)` ‚Äî SHA-256 of raw 32-byte Ed25519 public key
- `exportPrivateKeyPem(key)` ‚Äî export private key as PEM string
- `exportPublicKeyPem(key)` ‚Äî export public key as PEM string

### Constitution

- `loadConstitution(path)` ‚Äî load and parse YAML constitution
- `parseConstitution(data)` ‚Äî parse raw object to typed Constitution
- `validateConstitutionData(data)` ‚Äî validate structure, returns error list
- `verifyConstitutionSignature(constitution, publicKey)` ‚Äî verify Ed25519 signature
- `computeFileContentHash(path)` ‚Äî SHA-256 content hash of a file

### Authority Evaluator

- `evaluateAuthority(action, params, constitution)` ‚Äî 4-tier policy cascade evaluation
- `normalizeAuthorityName(name)` ‚Äî normalize action name per Appendix D

### Receipt

- `generateReceipt(params)` ‚Äî create a complete receipt with hashes and fingerprints
- `signReceipt(receipt, privateKey, signedBy)` ‚Äî Ed25519 sign a receipt
- `computeFingerprints(receipt)` ‚Äî compute 16-hex and 64-hex fingerprints
- `computeFingerprintInput(receipt)` ‚Äî compute the 16-field pipe-delimited input (checks_version >= 8) or 14/12-field for older receipts
- `SPEC_VERSION` ‚Äî protocol version (`"1.3"`)
- `CHECKS_VERSION` ‚Äî checks schema version (`"8"`)
- `TOOL_VERSION` ‚Äî package version string (`"sanna-ts/1.3.0"`)

### Verifier

- `verifyReceipt(receipt, publicKey?)` ‚Äî verify schema, signature, fingerprint, content hashes, status consistency, and timestamp

### Types

`Constitution`, `Boundary`, `HaltCondition`, `TrustTiers`, `TrustedSources`, `ConstitutionSignature`, `Provenance`, `AgentIdentity`, `Invariant`, `EscalationTargetConfig`, `EscalationRule`, `AuthorityBoundaries`, `AuthorityDecision`, `AuthorityDecisionType`, `BoundaryType`, `Receipt`, `CheckResult`, `ReceiptSignature`, `Enforcement`, `ConstitutionRef`, `VerificationResult`, `SannaKeypair`, `KeyObject`, `ReceiptParams`

## Cross-Language Compatibility

All hashing, fingerprinting, and signing operations are verified against golden fixtures from the [Sanna protocol spec](https://github.com/sanna-ai/sanna-protocol). Receipts generated by the [Python SDK](https://github.com/sanna-ai/sanna) verify correctly with this TypeScript SDK, and vice versa.
